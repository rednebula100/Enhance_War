# 강화대전 (Enhance_War) — 마스터 스펙

> 이 문서가 유일한 기준 문서. v1~v7 개별 파일들은 이 문서로 통합 완료, 더 이상 참고하지 말 것. 앞으로 규칙이 바뀌면 새 버전 파일을 만들지 말고 **이 문서를 직접 수정**할 것 (v1→v7처럼 덮어쓰면서 예전 내용이 사라지는 사고를 막기 위함).

## 0. 한눈에 보는 핵심 원칙
- 화폐 둘: **코인**(매치 리셋용) / **돈, 원 단위**(계정 영구 누적)
- 검이 망가지는 두 가지 별개 이벤트는 절대 혼동 금지 (§3 참고)
- 카드 25종 확정, 패시브가 더 많음(19:6)
- 7성 카드는 의도적으로 압도적으로 강하고 압도적으로 희귀함
- 라운드 수 제한 없음, HP 0 되는 쪽이 패배

## 1. 기술 스택
- 실시간 매치: Node.js + Express + Socket.io, Render 무료 티어

## 4. 강화 메커니즘
```
강화 비용(n) = 10 * 1.25^n          // n = 현재 단계
강화 성공률(n) = 5% + 90% * 0.88^n  // 감쇠 완만화: 0.82→0.88  // 초반 급락, 후반 완만
```
- 콤보: 연속 성공 횟수. **성공률에는 영향 없음**, 오직 "판매가치"에만 반영 (단, "노련한 감각" 카드는 예외로 콤보가 성공률에 영향을 주게 만듦 — §6 카드 목록 참고)
- 판매(cash out): 언제든 현재 검을 즉시 코인으로 환전 가능. 판매하면 검 단계 0, 콤보 0으로 리셋 (강화 시도 실패와 결과는 같지만 트리거가 다름 — 플레이어가 "내가 직접" 누르는 행동임)
```
판매가치(level, combo) = (level^1.7 * 18) * (1 + combo * 0.15)  // 지수 곡선: 초반 낮고 후반 급등
```

## 5. 전투(교전) 시스템
- 라운드 수 제한 없음. **HP가 먼저 0이 되는 쪽이 패배**
- 양쪽 시작 HP: **100**
```
ATK(level) = level * 10
DUR(level) = 50 + level * 15

// 누가 먼저 깨지는지 판정 (DUR 기반)
ticksA = ceil(DUR_A / ATK_B)
ticksB = ceil(DUR_B / ATK_A)
→ ticks가 더 적은 쪽이 먼저 파괴됨 → 그 쪽이 HP 피해를 입음
→ ticksA === ticksB면 무승부, 양쪽 피해 없음

// 피해량 (라운드 지날수록 증가, 캡 없는 무제한 라운드를 끝내기 위한 장치)
HP 피해 = 승자ATK * 0.3 * damageMultiplier(round)
damageMultiplier(round) = 1 + (round - 1) * 0.15
```
- 검 파괴 후 처리는 §3의 (B) 그대로: 단계 유지, 내구도만 회복
- ★주의: 양쪽 다 level 0(ATK 0)이면 ticks가 무한대라 무승부 무한 루프 가능성 있음 — 안전장치 필요(예: 일정 라운드 이상 양쪽 강화 0단계면 강제 무승부 처리 등, 확인 필요)

## 6. 매칭 — 봇 위장
- "대결 시작" → 대기열 → **5~8초** 안에 실제 유저 없으면 봇 자동 대체
- 봇은 가짜 닉네임 풀 사용(§9), 강화 시도 타이밍은 랜덤 지연(쿨타임 끝나고 0.2~1.5초 사이)으로 사람처럼 행동
- 서버 내부적으로만 `isBot` 플래그, **클라이언트로는 절대 전송 안 함**
- 랭킹 집계 시 봇 매치 제외 (코인/돈 보상은 동일하게 지급)

## 7. 카드 시스템

### 7-1. 획득/보유/거래
- 카드는 **오직 라운드 종료 후 상점에서 구매**. 강화 성공시 확률 드랍 같은 거 없음
- 손패 최대 **8장**
- 상점 UI: 하스스톤 전장 스타일. 상단 가로 진열대(상점 레벨에 맞는 희귀도 풀에서 추첨) → 아래로 드래그하면 구매. 손패 카드를 위로 드래그하면 판매
- 상점 기능: **리롤**(코인 내고 진열대 갈아엎기), **레벨업**(코인 내고 상점 레벨 ↑, 더 높은 희귀도 등장), **얼리기**(진열대 유지, **라운드당 2회** 제한)

## 10. 화면 흐름 & UI
로그인(구글) → 메인 메뉴 → 매칭 대기(5~8초) → 대결 화면 → [라운드 종료마다] 상점 화면 → 결과 화면

- **대결 화면**: 좌우 거울 구도(내 검/상대 검), 중앙 상단에 양쪽 HP바 + 라운드 번호 + 타이머, 각 진영에 강화단계/ATK/DUR/코인/판매가치, 강화·판매 버튼, **손패 8슬롯 상시 노출**(대결 화면에서도)
- **상점 화면**: 상단 진열대 4장, 리롤/레벨업/얼리기 버튼, 하단 손패(드래그 구매/판매)
- **결과 화면**: 최종 검 모습, 강화단계, 개인 최고 도달 단계(기록), 이번 매치 획득한 돈(원), 재매칭/메인으로 버튼
- **로그인 화면**: 구글 로그인 버튼만 (구글 Authentication 공개용 앱 이름: "강화대전")
- **메인 메뉴**: 닉네임/보유 돈/대결 시작/랭킹보드(보류)/카드 컬렉션(보류)/설정

## 13. 인게임 업그레이드 시스템

### 13-1. 스탯 업그레이드 (상점 화면)

상점 패널 우측 -> 버튼으로 카드 진열대와 업그레이드 패널을 슬라이드 전환.

| 업그레이드 | 효과 | MAX | 가격 |
|---|---|---|---|
| 성공률 부스트 | +5%p/Lv | +20%p (Lv.4) | 200 * 2^Lv 코인 |
| 쿨타임 감소 | -10%/Lv | -40% (Lv.4) | 150 * 2^Lv 코인 |

- STAT_SUCCESS_BASE_COST:200, STAT_SUCCESS_GROWTH:2, STAT_SUCCESS_PER_LV:0.05, STAT_SUCCESS_MAX_LV:4
- STAT_CD_BASE_COST:150, STAT_CD_GROWTH:2, STAT_CD_REDUCE_PER_LV:0.10, STAT_CD_MAX_LV:4
- sword.statUpgrades = { successLv, successBonus, cooldownLv, cooldownMul } (매치별 초기화)
- 성공률 보너스 _passiveMods() 경유, 쿨타임 감소 _enhanceCooldown()에서 cd *= cooldownMul

### 13-2. 방지권 (대결 화면)

강화 실패 시 검 파괴 1회 방지. MAX 7개.

- n개 보유 시 다음 1개 가격: 100 * (2^(n+1) - 1)
  - 0->1: 100c, 1->2: 300c, 2->3: 700c, 3->4: 1500c, ...
- qty개 일괄 구매 총액: 100 * (2^(n+qty+1) - 2^(n+1) - qty)
- 소모 우선순위: 보험증서(c014) > 방지권 > 일반 파괴
- 소모 시: 레벨/콤보 유지, 코인 환불 없음, enhance_result에 shieldConsumed:true 포함
- SHIELD_BASE_COST:100, SHIELD_MAX_COUNT:7
